# 模块变更记录

本目录按微服务维度记录功能、接口、配置、事件、表结构相关改动。

## 记录规则

- 每次改动一个模块时，都要在对应服务目录下新增一条变更记录。
- 文件命名格式：`YYYY-MM-DD-序号-变更主题.md`。
- 一次需求如果影响多个模块，应分别记录到各自模块目录，不要混写在一个文件里。
- 如果改动涉及数据库 DDL，必须同时补充对应 `sql/postgresql/<service>/changes/` 下的 SQL 文件，并在文档里写清楚影响表、执行顺序和回滚注意事项。
- 如果本次模块改动没有 DDL，也要在变更文档里明确写出“本次无 DDL 变更”。

## 推荐内容

每条记录建议包含：

1. 改动背景
2. 涉及模块 / 接口 / 事件 / 配置
3. 领域影响
4. 数据库变更说明
5. 测试与验证结果
6. 回滚或兼容性注意事项
